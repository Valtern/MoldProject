# MoldGuard User Manual (Desktop & Mobile)

This manual is intended for end users who want to use the MoldGuard web application hosted at [https://moldymoldbase.web.app/](https://moldymoldbase.web.app/). This document covers the complete guide for both the **Desktop** version (Part A) and the **Mobile** version (Part B).

MoldGuard is a web-based monitoring system designed to help monitor mold risk in rooms. After logging in, users can view the dashboard, check sensors, monitor devices, read analytics, open the About Us page, adjust settings, and log out securely.

---

# Part A: Desktop Guide

---

## A1. How to Open the Web App

1. Open a browser on your laptop or desktop.
2. Go to [https://moldymoldbase.web.app/](https://moldymoldbase.web.app/).
3. Wait for the login page to appear.
4. Enter the email and password provided by the admin.
5. Click **Sign in** to enter the dashboard.

Notes:
- No installation is required.
- The app does not need to be run locally by the customer.
- Make sure your internet connection is active as all data is read from the cloud.

## A2. Login, Create Account, and Forgot Password

### Login
1. Enter your email in the **Email** field.
2. Enter your password in the **Password** field.
3. Click **Sign in**.
4. If the credentials are correct, you will be redirected to the dashboard.

### Create Account
1. On the login page, click **Create account**.
2. Fill in the required registration form.
3. Follow the instructions until the account is successfully created.

### Forgot Password
1. On the login page, click **Forgot password?**.
2. Enter your registered email address.
3. Follow the password recovery instructions sent to your email.

### Suggested Screenshots
- `assets/user-manual-create-account.png` — registration page.
- `assets/user-manual-forgot-password.png` — forgot password page.

## A3. Main Layout After Login (Desktop)

After logging in, the web app displays a sidebar on the left and the main content area on the right. The available menus are **Dashboard**, **Sensors**, **Device**, **Analytics**, **About Us**, and **Settings**.

### How to Read the Layout
1. The left sidebar is used to navigate between pages.
2. The center area displays the content of the currently open page.
3. The bottom of the sidebar shows the user account and logout button.
4. The language button is located at the top to switch the web language.

### Suggested Screenshots
- `assets/user-manual-desktop-dashboard.png` — desktop main view.

## A4. Using the Dashboard (Desktop)

The Dashboard is the main summary page for quickly viewing room conditions.

### Steps
1. After logging in, make sure you are on the **Dashboard** page.
2. View the temperature, humidity, light, and mold risk cards.
3. Check the room status displayed at the top.
4. Use this page as a quick summary before opening other menus.

### Things to Note
- The dashboard helps users understand current conditions without reading raw data.
- If conditions change, they are usually visible on the value cards and risk status.

### Suggested Screenshots
- `assets/user-manual-dashboard.png` — main dashboard.

## A5. Using Sensors (Desktop)

The **Sensors** page is used to monitor sensors connected to the system.

### Steps
1. Click the **Sensors** menu on the sidebar.
2. View the list of available sensor nodes.
3. Check the temperature, humidity, and status of each sensor.
4. If any sensor shows a warning or offline status, check the physical device in the field.

### On Desktop
- Sensors are displayed as easy-to-read cards.
- Users can compare multiple sensors at once.

### Suggested Screenshots
- `assets/user-manual-sensors-desktop.png` — Sensors page.

## A6. Using Device (Desktop)

The **Device** page is used to view connected devices and check their status.

### Steps
1. Click the **Device** menu.
2. View the list of registered devices.
3. Check the device ID, room name, Wi-Fi signal, and online/offline status.
4. Use this page to verify that devices are still active.

### Edit and Delete Features
1. If an **edit** button is available, click it to update the room name or device ID.
2. If a **delete** button is available, click it to remove a device or room that is no longer in use.
3. Confirm the action to prevent accidental data deletion.

## A7. Using Analytics (Desktop)

The **Analytics** page is used to read risk reports and data trends.

### Steps
1. Click the **Analytics** menu.
2. View the list of predictive alerts on the left side.
3. Check the trend charts and data stream on the right side.
4. Use the time filter if available, e.g., 24H, 7D, or 30D.
5. If there is an important alert, read the risk details before dismissing it.

### Purpose of This Page
- Helps users see data patterns, not just current values.
- Useful for evaluating room conditions and taking follow-up action.

The **About Us** page provides information about the product, technology, development team, supervising lecturers, and institution.

## A9. Using Settings (Desktop)

The **Settings** page is used to change system settings according to the user's needs.

### Steps
1. Click the **Settings** menu.
2. Change the thresholds if your account has access.
3. Set up the notification email if the feature is used.
4. Choose a display mode such as Light, Dark, or System.
5. Enable or disable the ripple effect if needed.

2. Click to select the available language, e.g., `ID` or `EN`.

### Changing Theme
1. Click the theme icon in the top-right corner.
2. Choose Light, Dark, or System.
3. The web app will adjust the display according to your choice.

## A11. Logout (Desktop)

### Steps
1. Look at the bottom of the sidebar or the account menu.
2. Click the logout button.
3. Confirm if a warning dialog appears.
4. After logging out, you will be returned to the login page.

### Important
- Log out when you are done using the web, especially on shared devices.
- Do not leave your account open on public browsers.

### Suggested Screenshots
- `assets/user-manual-logout-confirm.png` — logout dialog.

## A12. Quick Usage Flow (Desktop)

1. Open the web app in a browser.
2. Log in or create an account.
3. Check the Dashboard for a condition summary.
4. Open Sensors or Device to see device details.
5. Open Analytics to read reports and trends.
6. Open Settings to change preferences if needed.
7. Log out when finished.

## A13. Troubleshooting (Desktop)

- If the page does not open, check your internet connection and refresh the browser.
- If login fails, verify your email and password.
- If the language does not change, click the language option again.
- If data does not appear, wait a moment as the web reads realtime data from the cloud.
- If the edit or delete button does not appear, your account may not have access for that action.

# Part B: Mobile Guide

---

## B1. Differences Between Desktop and Mobile Layout

| Aspect | Desktop | Mobile |
|---|---|---|
| Main navigation | Sidebar on the left | Bottom navigation bar (tabs at bottom) |
| Profile menu | Bottom of sidebar (email + logout button) | Profile button in the top-right corner (dropdown) |
| About Us & Settings | Sidebar menu | Hidden inside the profile dropdown (top-right corner) |
| Content layout | Multi-column (cards side by side) | Single column (cards stacked vertically) |
| Language button | Top of sidebar | Top-left corner next to the logo |

## B2. How to Open on Mobile

1. Open a browser on your smartphone (Chrome, Safari, or another browser).
2. Type or paste **https://moldymoldbase.web.app/** in the address bar.
3. Wait for the login page to appear.
4. Enter your email and password.
5. Tap the **Sign in** button.

Tips:
- Add the page to your home screen so it can be accessed like a native app.
- Make sure your internet connection is stable as all data is read from the cloud in realtime.

## B3. Login, Create Account, and Forgot Password (Mobile)

### Login
1. Fill in the **Email** field with your registered email.
2. Fill in the **Password** field.
3. Tap **Sign in**.
4. If successful, you will enter the Dashboard page.

### Create Account
1. On the login page, tap **Create account**.
2. Fill in the registration form (name, email, password).
3. Follow the instructions until the account is successfully created.
4. Log in using the newly created account.

### Forgot Password
1. On the login page, tap **Forgot password?**
2. Enter your registered email.
3. Check your email inbox for password reset instructions.

## B4. Main Navigation on Mobile

After logging in, the main navigation appears as a **bottom navigation bar** at the bottom of the screen. Available tabs:

| Icon | Tab Name | Function |
|---|---|---|
| 🏠 | **Dashboard** | Realtime room condition summary |
| 📡 | **Sensors** | List of sensor nodes and data readings |
| ⚡ | **Device** | IoT device connection status |
| 📊 | **Analytics** | Risk reports, data trends, and predictive alerts |

### Profile Menu (Top-Right Corner)
Tap the **user icon/name** in the top-right corner to open a dropdown containing:
- **Username and email** (currently logged-in account information)
- **About Us** — project, team, and technology information
- **Settings** — threshold, notification, and appearance settings
- **Log out** — sign out of the account

## B5. Using the Dashboard (Mobile)

The Dashboard is the main page that appears immediately after login. On the mobile view, all content is arranged in a single vertical column.

### Components Displayed (top to bottom)
1. **Room header** — Displays the active room name and status (e.g., "lkj3test1 — Warning"). The status color indicates the danger level.
2. **Sensor cards** — Three cards displaying:
   - **Temperature** (in °C, status: Comfortable/Hot/Cold)
   - **Humidity** (in %, status: Humid/Dry/Normal)
   - **Light Level** (in lux, status: Bright/Dim/Dark)
3. **Mold risk gauges** — Two circular gauges:
   - **General Mold Risk** (general mold risk percentage)
   - **Black Mold Risk** (black mold / Stachybotrys risk percentage)
   - Each displays a status: Low Risk, Medium Risk, or High Risk.
4. **Humidity History** — Humidity history chart (default: last 24 hours) with a safe range line (Safe 40-60%).
5. **Appliance Control** — Device controls:
   - Choose **Auto** or **Manual** mode.
   - Toggle **Exhaust Fan** (On/Off).
   - Toggle **Dehumidifier** (On/Off).
   - **Trigger Thresholds** section (can be expanded/collapsed with a tap).
   - Note: overrides are sent to the ESP32 on its next data push cycle.

### How to Read Room Status
- **Safe / Low Risk** (green): Conditions are safe, no action needed.
- **Warning / Medium Risk** (yellow/orange): Conditions are starting to be risky, monitor more frequently.
- **Critical / High Risk** (red): Dangerous conditions, take immediate action (ventilation, dehumidifier, check device).

## B6. Using Sensors (Mobile)

1. Tap the **Sensors** tab on the bottom navigation bar.
2. View the list of sensor nodes connected to your account.
3. Check the temperature, humidity, and light readings from each node.
4. Note the status of each sensor (active, warning, or offline).

On the mobile view, sensor cards are displayed stacked vertically for easy scrolling.

## B7. Using Device (Mobile)

1. Tap the **Device** tab on the bottom navigation bar.
2. View the list of registered IoT devices.
3. Check the following information on each device card:
   - **Device ID** — unique device identifier.
   - **Room name** — room connected to the device.
   - **Wi-Fi Signal** — connection signal strength.
   - **Status** — Online or Offline.
4. If available, tap the **Edit** button to change the room name or device ID.
5. If available, tap the **Delete** button to remove a device/room no longer in use.

## B8. Using Analytics (Mobile)

1. Tap the **Analytics** tab on the bottom navigation bar.
2. View the list of **Predictive Alerts** containing risk warnings.
3. Check the **Trend Data** charts to see humidity and temperature patterns.
4. Use the time filter if available:
   - **24H** — last 24 hours of data.
   - **7D** — last 7 days of data.
   - **30D** — last 30 days of data.
5. Tap the **Export CSV** button to download data as a CSV file (useful for further analysis or auditing).
6. Alerts that have been read can be dismissed to keep the list clean.

## B9. Accessing About Us (Mobile)

1. Tap the **profile icon** (number/name) in the top-right corner of the screen.
2. In the dropdown, tap **About Us**.
3. This page displays:
   - MoldGuard project summary.
   - Brief explanation of system features and approach.
   - **Technology Highlights** — IoT Sensor Network, AI-Powered Prevention, Proactive Control, and Cloud for real-time access.
   - **Development Team** (Group 2) — list of team members and their roles.
   - **Supervising Lecturers** — project supervisors.
   - **Institution** — Study Program and Polytechnic.

## B10. Accessing Settings (Mobile)

1. Tap the **profile icon** in the top-right corner.
2. Tap **Settings** in the dropdown.
3. The Data Management page will open containing:

### Threshold Configuration
- **General Mold Thresholds:**
  - Safe Limit (% RH) — humidity below this value is considered safe.
  - Critical Limit (% RH) — humidity above this value triggers critical alerts.
- **Toxic Black Mold Thresholds:**
  - Safe Limit (% RH) — safe limit for black mold risk.
  - Critical Limit (% RH) — critical limit (maximum configurable: 90% RH).
  - Note: 90% RH is the biological threshold for *Stachybotrys chartarum* growth. The system does not allow settings above this value.

### Enable Email Alerts
- Toggle to enable/disable automatic email notifications.

### Notification Preferences
- Choose the notification levels you want to receive:
  - **Low Risk** — receive all emails.
  - **Medium Risk** — medium risk and above only.
  - **High Risk** — high risk only.
- You can select one, multiple, or all levels.

### Appearance
- **Light** — light display mode.
- **Dark** — dark display mode.
- **System** — follows device settings.

### Click Ripple Effect
- Toggle to show or hide the wave animation when tapping elements.

### Saving Settings
- After changing settings, you **must** tap the **Save Changes** button (green, bottom-right corner).
- If not saved, changes will not take effect.

## B11. Changing Language (Mobile)

1. Tap the language button in the top-left corner (next to the MoldGuard logo), usually labeled **EN** or **ID**.
2. Select the desired language.
3. All interface text will change according to your selection (Bahasa Indonesia or English).

## B12. Logout (Mobile)

1. Tap the **profile icon** in the top-right corner.
2. Tap **Log out** (in green) in the dropdown.
3. A confirmation dialog **"Log out?"** will appear with buttons:
   - **No** — cancel logout.
   - **Yes** — confirm logout.
4. After confirming, you will be returned to the login page.

Important:
- Always log out when using shared devices.
- Do not leave your account open on public browsers.

## B13. Quick Usage Flow (Mobile)

1. Open **https://moldymoldbase.web.app/** in a mobile browser.
2. **Log in** using your email and password.
3. Check the **Dashboard** — view temperature, humidity, light, and mold risk status.
4. Open **Sensors** — check sensor node status.
5. Open **Device** — make sure IoT devices are still online.
6. Open **Analytics** — read predictive alerts and data trends.
7. Open **Settings** (via profile dropdown) — adjust thresholds if needed.
8. **Log out** when finished.

## B14. Troubleshooting (Mobile)

| Issue | Solution |
|---|---|
| Page does not open | Check internet connection, try refreshing the browser. |
| Login fails | Make sure email and password are correct. Try resetting password if forgotten. |
| Data does not appear | Wait a few seconds as data is read in realtime from the cloud. Try refreshing. |
| Language does not change | Tap the language option again. Make sure the page has finished loading. |
| Appliance toggle not responding | Make sure **Manual** mode is active. Override takes effect on the next ESP32 push. |
| Settings not saved | Make sure you tapped **Save Changes** before leaving the page. |
| Display too small | Make sure browser zoom is at 100%. Try rotating to landscape if needed. |

## B15. Tips for Mobile Usage

- **Bookmark** the page for easy access later.
- **Add to Home Screen** via the browser menu so MoldGuard can be accessed like a native app.
- Use **Dark mode** at night for eye comfort.
- Check the dashboard at least **once a day** to keep room conditions monitored.
- Enable **email alerts** to receive automatic notifications without having to open the app.
